## 4. Description of AI Use

The core idea for BuildBridge SA — a surplus construction materials marketplace targeting South African township builders — was conceived by me based on observations about construction waste and the affordability challenges facing informal builders in Cape Town. I documented this idea in a detailed project blueprint before any AI assistance was used for technical implementation.

I used Claude Code as a terminal-based AI coding assistant throughout the development phase. I provided Claude Code with precise technical specifications for each component — defining the data structure, the calculation formulas for the Smart Estimator, the state management architecture, and the visual design language (colour palette, typography, animation behaviour). Claude Code then suggested code patterns and component structures based on these instructions.

However, every output required substantial direction and review on my part. I specified the exact business logic for the Smart Project Estimator (the calculateWall function using South African standard brick sizing), defined the product data schema, determined the UX flow, chose the animation approach (IntersectionObserver, clip-path reveals, SVG stroke-dashoffset), and made all design decisions. When Claude Code's suggestions did not match my requirements, I corrected and redirected it.

For the business and UX documentation, I used Claude (claude.ai) as a sounding board to help me structure my initial ideas into formal frameworks (Business Model Canvas, user personas). The content — the specific business model, the South African market context, the personas of Sipho and Sarah, and the research into disposal costs and retail price benchmarks — came from my own research and thinking.

## 6. Critical Reflection on My Approach

**Word Count:** ~280 words

My decision to use Claude Code as an AI coding assistant was driven by a desire to produce a technically sophisticated application that genuinely demonstrates the full range of eCommerce functionality, while keeping my focus on the design thinking, business model, and creative problem-solving that the project is really assessing.

**What worked well:** Claude Code was highly effective as a "pair programmer" for boilerplate-heavy tasks — generating repetitive component structures, applying Tailwind CSS classes consistently, and translating my precise specifications into working React code quickly. This allowed me to spend more time on the aspects that required genuine human thinking: the business model, the UX design, the Smart Estimator concept, and the visual design direction.

**What required the most human input:** The Smart Project Estimator was the feature that required the most original thinking from me. The idea of treating the platform as a digital quantity surveyor — automatically calculating material requirements from physical dimensions and matching them to live surplus inventory — is not something AI suggested. I conceived this feature specifically to address the real-world problem that informal builders face (purchasing errors due to lack of technical knowledge). I then had to specify the exact formula, the product matching logic, the bundle discount mechanism, and the UI presentation in precise detail before the AI could assist with implementation.

**Limitations I encountered:** Claude Code occasionally produced code that did not match the exact hook signatures or import paths used elsewhere in the project, requiring me to read, understand, and correct the output before it could be integrated. This reinforced that AI is a tool that amplifies a developer's ability — it does not replace the need to understand what you are building.

**What I learned:** Directing an AI coding assistant effectively requires a stronger understanding of software architecture, not a weaker one. The quality of the output is entirely determined by the quality and specificity of the instructions — which is a design and engineering skill in itself.
